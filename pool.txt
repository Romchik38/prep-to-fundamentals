Instance -> Class -> Factory -> Pool

Фабритка:
  - объектов
  - прототипов
  - классов
  - функций
  - фабричные методы
  - factorify

Для прототипов делается специальная функция типа logable:
  - в нее передавать объект с условием
  - на выходе она будет отдавать прототип.
  - при создании нового инстанса и передачи ему значений будет проходить проверка.

Для класса также делается функция logable

Для функций делается своя фабрика:
  - в нее передается значение
  - на основе этого значения определяется поведение функции
  - возвращается функция, которая будет вести себя определенным образом.

Фабричные методы:
  - в классах создается статический метод.
  - при вызове этого метода создается новый инстанс

Factorify
  - функция, в замыкание которой передается класс.
  - функция возвращает ссылку на функцию из которой потом будут создаваться инстансы
--------------------------------------------------------
Pool
  - из пула берутся элементы, если он вызывается как функция pool()
  - в пул калдутся элементы, если при вызове передается аргумент pool(item)
  - при запуске проверяются созданы ли элементы в пуле. если нет, то создаются.
  - если передан item, то он пушается в пул
  - если не передан элемент, то из пула далется pop или новый элемент, если в пуле уже пусто
